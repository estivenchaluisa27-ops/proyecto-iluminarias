2. Firebase MCP #1 (lab-redes-turnos) y #2 (iluminariasuce)
Aspecto	Detalle
Estado	Parcialmente funcional — errores de schema en tools que retornan arrays
Config	npx firebase-tools experimental:mcp con env FIREBASE_PROJECT
Pruebas exitosas	get_environment, list_projects (4 proyectos), get_project, get_sdk_config (web), firestore_list_collections (coleccion luminarias), firestore_get_database, firestore_get_security_rules, firestore_validate_security_rules (OK), firestore_get_database
Datos descubiertos	Proyecto iluminariasuce activo, Firestore en nam5, free tier, edition STANDARD, database (default), sin PITR, sin delete protection. Reglas de Firestore abiertas con expiración 2026-07-22.
Errores encontrados (6 errores)
1. firebase_list_apps → Error: Invalid tools/call result: expected record, received array. El schema del MCP define el resultado como objeto pero retorna array. Root cause: bug de firebase-tools en el output schema del MCP.
2. auth_get_users → Mismo error: expected record, received array. El tool interna retorna un array de usuarios pero el schema espera un objeto.
3. functions_list_functions → Mismo error: expected record, received array. Igual root cause.
4. firestore_list_documents → Error: data/documents/0/fields/etiqueta/nullValue must be string. El campo etiqueta en Firestore tiene valor null y el schema del MCP no acepta nullValue como tipo válido en la posición esperada. Root cause: bug en el output schema que no maneja NULL_VALUE enum correctamente.
5. firestore_query_collection → Error 400: read_time cannot be in the future. El MCP pasa un timestamp futuro por defecto. Workaround: omitir filtros que dependen de read_time.
6. Configuración duplicada: Ambos servers Firebase apuntan al mismo proyecto iluminariasuce (el env FIREBASE_PROJECT=lab-redes-turnos del primer server es ignorado porque firebase.json override a iluminariasuce). Impacto: el MCP #1 es redundante.
Oportunidades de mejora
- Firebase Apps no registradas: Detected App IDs: <NONE>. Necesitas crear una Web App en Firebase para desbloquear hosting deploys y SDK config automático.
- Billing no habilitado: Billing Enabled: No. Sin billing no puedes usar Cloud Functions, Cloud Storage (post free tier), ni PITR.
- Reglas de Firestore peligrosamente abiertas: allow read, write: if request.time < timestamp.date(2026, 7, 22) — se bloquearán el 22 de julio (11 días). Hay que escribirlas antes.
- firestore_create_index disponible: La coleccion luminarias necesita indices para queries por estado. El tool existe y funciona.
- firebase_create_app disponible: Puedes crear la Web App faltante via MCP.
- MCP #1 redundante: Eliminar el server firebase o cambiarle el FIREBASE_PROJECT a lab-redes-turnos real (crear directorio sin firebase.json que override).
